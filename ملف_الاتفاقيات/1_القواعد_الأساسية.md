# Dork App - اتفاق العمل مع Claude
## الجزء 1️⃣: القواعد الأساسية

---

## ⚠️ قاعدة مهمة:
**قبل رفع أي كود للـ GitHub:**
1. سيتم اختبار الكود محلياً أولاً
2. التأكد من عدم وجود أخطاء
3. التأكد من أن الميزة تعمل بشكل صحيح
4. عرض النتائج عليك للمراجعة
5. فقط بعد موافقتك "OK" يتم الرفع للـ GitHub

---

## الاتفاقيات الأساسية:

### 1. طريقة العمل

❌ **خطأ:** تعطيني مهمة وأنا أبدأ بسرعة بدون استشارة
✅ **الصحيح:** خطوة بخطوة، استشارة قبل التنفيذ، بدون تسرع أو تخريب
✅ كل خطوة أعرضها عليك قبل ما أكملها

---

### 2. قبل أي تعديل

❌ **خطأ:** أعدل الكود مباشرة بدون backup أو خطة
✅ **الصحيح:** أخذ نسخة قديمة (backup) قبل أي تعديل
✅ عرض الخطة على المستخدم أولاً
✅ انتظار الموافقة "OK" قبل التنفيذ

---

### 3. الرفع للـ GitHub

❌ **خطأ:** رفع الكود بدون موافقة أو بـ force push بدون إذن
✅ **الصحيح:** الرفع بعد الموافقة فقط
✅ مع رسالة commit واضحة ومفصلة
✅ بدون force push بدون إذن صريح

---

### 4. الاختبار

❌ **خطأ:** رفع الكود بدون اختبار محلي
✅ **الصحيح:** التأكد من عدم وجود أخطاء قبل الرفع
✅ شرح النتائج بوضوح للمستخدم

---

### 5. اتفاقية الترابط والإدارة (مهم جداً)

❌ **خطأ:** إضافة ميزة للعميل/الصالون بدون تفعيلها في لوحة الإدارة
✅ **الصحيح:** **القاعدة الذهبية:** أي إضافة مهمة (PIN، بريد، رقم جوال) = يجب تكون في Admin Panel + 2-way sync
✅ الإضافات المهمة أضيفها في التطبيق + الإدارة معاً بدون سؤال
✅ الإضافات البسيطة (ألوان، خطوط) أسأل أولاً قبل التنفيذ
✅ في حالة الشك → اسأل بدل الافتراض

---
