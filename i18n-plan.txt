----------------------------------------------------------------
١. الهدف من الخطة
----------------------------------------------------------------

العمال الأجانب في الصالونات السعودية يحتاجون فهم لوحة الصالون:
  - الباكستانيون والهنود والبنغاليون  → الأردية (Urdu)
  - الأتراك                            → التركية (Turkish)
  - المالك السعودي                      → العربية (Arabic) — الحالي
  - توسع مستقبلي                        → الإنجليزية (English)

القاعدة الأساسية:
  - واجهة العميل  → عربي + إنجليزي فقط
  - لوحة الصالون  → عربي + إنجليزي + أردي + تركي


----------------------------------------------------------------
٢. الأدوات المستخدمة
----------------------------------------------------------------

  المكتبة الرئيسية : i18next
  تكامل React      : react-i18next
  التحميل الكسول  : i18next-http-backend

  السبب:
  - مكتبات مجانية ومفتوحة المصدر
  - الأكثر استخداماً عالمياً لهذا الغرض
  - لا تزيد حجم الـ Bundle (الملفات في public/)
  - تدعم RTL و LTR بشكل مدمج
  - التحميل عند الطلب فقط (Lazy Loading)


----------------------------------------------------------------
٣. هيكل الملفات المطلوب
----------------------------------------------------------------

  /public/locales/
  ├── ar/
  │   └── common.json       ← عربي (العميل + الصالون)
  ├── en/
  │   └── common.json       ← إنجليزي (العميل + الصالون)
  ├── ur/
  │   └── common.json       ← أردي (الصالون فقط) ★
  └── tr/
      └── common.json       ← تركي (الصالون فقط) ★

  ★ هذان الملفان لن يُحمَّلا أبداً في واجهة العميل

٤. المراحل التفصيلية

  ══════════════════════════════════════════════════════
  المرحلة ١ — البنية التحتية (لا تغيير في الواجهة)
  ══════════════════════════════════════════════════════
  المدة  : يوم واحد
  الخطر  : صفر — التطبيق لن يتأثر أبداً

  المهام:
  [ ] تثبيت: npm install i18next react-i18next i18next-http-backend
  [ ] إنشاء ملف إعداد: src/i18n.js
  [ ] إنشاء مجلد /public/locales/ بالهيكل أعلاه
  [ ] إنشاء JSON فارغة كـ placeholder لكل لغة
  [ ] إضافة <I18nextProvider> في main.jsx
  [ ] التأكد من أن التطبيق يعمل 100% بنفس الشكل
  [ ] إضافة زر اللغة في SalonDrawer (مخفي حتى تكتمل الترجمة)

  الاختبار: التطبيق يعمل كما كان، صفر تغييرات مرئية

  ══════════════════════════════════════════════════════
  المرحلة ٢ — ترجمة الصالون (الأولوية القصوى)
  ══════════════════════════════════════════════════════
  المدة  : ٤ أيام
  الخطر  : منخفض (مكوّن واحد في كل مرة)

  اليوم ١: SalonDrawer
    - قائمة التنقل الرئيسية
    - أسماء الأقسام
    - نصوص الأزرار
    السبب: أول شيء يراه العامل عند فتح التطبيق

  اليوم ٢: OwnerDash (لوحة التحكم)
    - عناوين البطاقات (مقبول / انتظار / مرفوض / الكل)
    - الإحصائيات
    - أزرار الإجراءات
    السبب: الأكثر استخداماً يومياً

  اليوم ٣: NotifPanel (قائمة الحجوزات)
    - تفاصيل الحجز
    - حالات الحجز
    - معلومات العميل
    السبب: العامل يحتاج يفهم من حجز وأي خدمة

  اليوم ٤: OwnerSettings (الإعدادات)
    - أقل أهمية للعمال
    - يُترجم أخيراً

  ══════════════════════════════════════════════════════
  المرحلة ٣ — ترجمة العميل (اختياري)
  ══════════════════════════════════════════════════════
  المدة  : يومان
  الخطر  : منخفض

  اللغات: عربي + إنجليزي فقط
  الفائدة: للمقيمين الأجانب الذين يحجزون
  ملاحظة: أقل أهمية — العملاء السعوديون لا يحتاجونه

  ══════════════════════════════════════════════════════
  المرحلة ٤ — الصقل والاختبار
  ══════════════════════════════════════════════════════

  [ ] مراجعة الترجمات من ناطق أصلي بالتركية

----------------------------------------------------------------
٦. قواعد الأمان (لا نتجاوزها أبداً)
----------------------------------------------------------------

  القاعدة ١: Arabic Fallback دائماً
    → i18next يعرض العربي تلقائياً إذا ما وُجدت ترجمة
    → التطبيق لا ينكسر أبداً بسبب ترجمة ناقصة

  القاعدة ٢: فرع git منفصل لكل مرحلة
    → فرع: feature/i18n-phase-1
    → فرع: feature/i18n-phase-2
    → إذا حدث خطأ نرجع للفرع السابق فوراً

  القاعدة ٣: مكوّن واحد في كل مرة
    → لا نترجم كل شيء دفعة واحدة
    → نختبر بعد كل مكوّن قبل الانتقال للتالي

  القاعدة ٤: الترجمات في public/ وليس في Bundle
    → لا تأثير على حجم ملف التطبيق
    → تُحمَّل عبر HTTP عند الحاجة فقط

  القاعدة ٥: لا نلمس وظائف التطبيق
    → نغيّر النصوص فقط
    → صفر تغيير في المنطق أو الأكواد


----------------------------------------------------------------
٧. المشاكل المتوقعة وحلولها
----------------------------------------------------------------

  المشكلة ١: التركية LTR (عكس العربية والأردية)
  الحل     : تبديل direction: rtl ↔ ltr ديناميكياً
             عند تغيير اللغة

  المشكلة ٢: خط الأردية ثقيل (Noto Nastaliq ~400KB)
  الحل     : تحميله كسولاً فقط عند اختيار الأردية
             <link> يُضاف ديناميكياً

  المشكلة ٣: النصوص داخل Toast و Callbacks
             مثل: toast$("✅ تم الحفظ")
  الحل     : نمرر مفتاح الترجمة بدل النص مباشرة
             toast$(t("saved_success"))

  المشكلة ٤: البنغاليون يتكلمون Bengali وليس Urdu
  الحل     : الأردية مفهومة لأغلبهم في السياق السعودي
             لكن يمكن إضافة bn.json لاحقاً إذا احتجنا

  المشكلة ٥: جودة الترجمات الآلية
  الحل     : نستخدم الترجمة الآلية كمسودة أولى
             ثم مراجعة إلزامية من ناطق أصلي


----------------------------------------------------------------
٨. ما لا يتغير أبداً
----------------------------------------------------------------

  ✗ لا تغيير في منطق الحجز أو قاعدة البيانات
  ✗ لا تغيير في التصميم أو الألوان
  ✗ لا تغيير في بنية المكوّنات
  ✗ لا تغيير في واجهة العميل (إلا في المرحلة ٣)
  ✗ لا إضافة لغات جديدة غير المتفق عليها
